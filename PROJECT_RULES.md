# Project Rules / 项目规则

---

## 🎯 Project Objectives / 项目目标

### English
- **Primary Goal**: Create educational algorithm implementations with comprehensive documentation
- **Vision**: Develop clear, well-documented code examples for learning purposes
- **Focus**: Algorithm education with practical demonstrations and step-by-step explanations

### 中文
- **主要目标**: 创建具有全面文档的教育性算法实现
- **愿景**: 开发清晰、文档完善的代码示例用于学习目的
- **重点**: 算法教育，包含实践演示和逐步解释

---

## 📝 Coding Standards / 编码规范

### English
- **Code Language**: All code, comments, docstrings, and variable names must be in English
- **Code Style**: Follow PEP 8 Python code standards
- **Naming Conventions**: Use snake_case for variables and functions, PascalCase for classes
- **Documentation Requirements**: Every function and class must have English docstrings
- **Code Readability**: Write clear, self-explanatory code
- **Error Handling**: Implement proper exception handling
- **Testing**: Include comprehensive test cases for all functionality

### 中文
- **代码语言**: 所有代码、注释、文档字符串、变量命名必须使用英文
- **代码风格**: 遵循PEP 8 Python代码规范
- **命名规范**: 使用snake_case命名变量和函数，PascalCase命名类
- **注释要求**: 每个函数和类必须有英文docstring
- **代码可读性**: 编写清晰、自解释的代码
- **错误处理**: 实现适当的异常处理
- **测试**: 为所有功能包含全面的测试用例

## 🔄 Workflow / 工作流程

### English
1. **Project Setup**: Create clear directory structure with comprehensive README
2. **Implementation**: Write clean, well-documented code following established standards
3. **Testing**: Include comprehensive test cases covering normal and edge cases
4. **Documentation**: Create detailed documentation with usage examples
5. **Demonstration**: Provide working examples and educational content
6. **Review**: Ensure code follows all established principles and standards
7. **Deployment**: Verify project runs without issues and is ready for use

### 中文
1. **项目设置**: 创建清晰的目录结构，包含全面的README
2. **实现**: 编写清洁、文档完善的代码，遵循既定标准
3. **测试**: 包含涵盖正常情况和边界情况的全面测试用例
4. **文档**: 创建包含使用示例的详细文档
5. **演示**: 提供工作示例和教育内容
6. **审查**: 确保代码遵循所有既定原则和标准
7. **部署**: 验证项目无问题运行并准备使用
